# ADM — Studio

ADM Studio is the environment where creators build the components that define worlds, entities, and experiences.

It provides tools for creating, packaging, simulating, and sharing content within the ADM ecosystem.

---

## Ven Creation

Creators can define **Vens**, the fundamental units of existence in ADM.

Types of Vens include:
- character Vens  
- object Vens  
- location Vens  
- event Vens  
- narrative Vens  

Each Ven contains:
- properties  
- behaviors  
- relationships  
- metadata  

Vens can be used individually or combined to form more complex systems.

---

## Packaging and Marketplace

Vens can be grouped into **packages** and shared through a marketplace.

Examples:
- “fancy table” (object set)  
- “sexy vampire pack” (character + thematic elements)  
- “vampire bar environment” (combined package)

Creators can:
- assemble Vens into reusable sets  
- publish and distribute packages  
- combine packages into larger environments  

This allows modular construction of worlds.

---

## World Construction

Creators build worlds by:
- selecting and combining Vens and packages  
- defining rules and structure  
- specifying required elements and procedural generation behavior  

Worlds can be composed from both fixed elements and generative systems.

---

## Simulation Tools

ADM Studio includes simulation tools for testing behavior.

Creators can:
- run rapid simulations of Vens and events  
- observe how systems interact  
- test outcomes before deployment  

Example:
A creator can define a complex object or event (e.g. a spell with multiple effects) and simulate how it behaves within a world context.

---

## VOX Integration

Creators can define **VOX packs** within the studio.

These determine:
- narrative voice  
- tone and style  
- how system output is expressed to players  

VOX packs can be applied across different worlds.

---

## VIZ Mode

**VIZ mode** allows creators to define visual styles.

Creators can:
- train or define visual representations  
- control how entities and environments appear  
- apply styles to worlds and player experiences  

VIZ operates similarly to VOX, but for visual output instead of narrative.

---

## Player Identity and Social Layer

Players maintain a persistent identity across the system.

- Each player has a Ven (their active identity)
- Their experiences are reflected in a **lookbook**

The lookbook contains:
- visual captures (“renders”) of the player’s Ven  
- generated from different VIZ styles  
- based on moments experienced across worlds  

This creates a record of how the player has existed and been perceived across different contexts.

---

## Summary

ADM Studio enables creators to:
- define entities (Vens)  
- package and distribute content  
- construct worlds  
- simulate behavior  
- define narrative (VOX) and visual (VIZ) expression  

It supports a modular, extensible ecosystem where worlds are built from shared components.
